int* const 的含义

  • int*：这是一个指向 int 类型数据的指针。
  • const 修饰指针本身：当 const 放在指针声明的尾部（即 int* const），表示指针本身不可改变，即指针一旦指向某个地址，就不能再指向其他地址。

完整解释：

在上面的代码中，p 是一个 指向 int 类型数据的常量指针。虽然我们可以通过 p 修改它指向的地址中的数据（例如 *p = 10;），但是不能修改 p 本身的指向（即不能让它指向别的地址）。

指针常量与常量指针的区别

  1. int* const p：
    • 指针是常量（p 不可改变）。
    • 但是指向的对象（即 *p）可以被修改。
  2. const int* p 或 int const* p：
    • 指向的对象是常量（*p 不可改变）。
    • 但是指针本身可以改变（p 可以指向其他地址）。
  3. const int* const p：
    • 指针和指向的对象都是常量：既不能改变 p 的指向，也不能修改 *p 的值。

二维数组传参为什么需要明确标注除第一维外的维度？

  • 这是因为 C++ 编译器需要知道每一行占据多少内存。在本例中，每一行 a[k]有 4 个 int 元素，所以：
    • 每行占据 4 * sizeof(int) 字节。
    • 如果不明确指定列数（即 4），编译器无法正确计算每行的偏移。

示例：

  • 编译器根据 **a[k] 的长度（即 4 个整数的大小）**来进行偏移。

因此，必须在函数声明中指定所有除第一维以外的维度，让编译器正确处理每行的偏移。

2. 符号表与静态符号映射

在PPT左下角，我们可以看到符号表的部分：

  • 符号表（Symbol Table）：

    • 符号表保存了程序中变量、函数等符号的名称和它们在内存中的地址。这些地址帮助编译器和链接器在程序中找到正确的函数或变量。

  • 符号名解析（Name Mangling）：

    • C++在符号表中将函数名称进行修饰（Name Mangling），以支持重载：
      • _Z4funci：这是表示**int func(int)**的修饰名称。
      • _Z4funcid：这是表示**int func(int, double)**的修饰名称。
    • Name Mangling 是必要的，因为 C++ 支持函数重载，不同的函数需要被区分开。经过修饰的符号名称会包含函数的参数类型信息，因此同名函数（不同参数）在符号表中不会冲突。

3. 内存布局与运行时环境（Runtime Environment）

右侧展示了典型C/C++程序的内存布局，分为：

  • Code 区段（R）：存放程序的指令，即代码本身。
  • Data 区段（RW）：存储全局变量和静态变量，这些变量在整个程序运行期间都存在。
  • Heap 堆区（RW）：用于动态内存分配，比如通过malloc或new分配的内存。
  • Stack 栈区（RW）：存储局部变量和函数调用的参数。

每个区域的细节：

Code 区段

  • 存放可执行代码和函数，比如func(int) 和 func(int, double)。
  • COST标签表示：函数调用会有性能开销，如栈帧的创建和参数传递。

Data 区段

  • 静态全局变量x会被存放在Data区。它的内存地址是0x02020202，这个地址在程序运行期间不会改变。

Heap 堆区

  • 动态分配的内存存储在Heap区，并且内存地址是0x03030303。
  • 动态内存需要手动管理，使用malloc/free或new/delete。

Stack 栈区

  • 局部变量d在栈区分配，栈区是**LIFO（后进先出）**结构，随着函数调用栈的增长和缩减而变化。变量d的地址在符号表中为0x04040404。

3. 函数调用的COST

  • PPT中的COST标签：
    • 每次函数调用都会有一定的性能开销：
      • 创建新的栈帧，存储局部变量和函数参数。
      • 在函数退出时，栈帧会被销毁。

4. 三个Questions（仅供参考）

Q1. 能否不降低可读性的前提下，降低COST？

答案：

  • 可以通过**内联函数（inline）**减少函数调用的开销，避免频繁的栈操作。
  • 使用模板函数（template），将代码直接展开在编译时。
  • 减少递归调用和优化参数传递方式（如通过引用或指针传递）。

Q2. 使用Lib需要注意什么？

答案：

  • ABI兼容性：库的二进制接口（ABI）必须与调用代码兼容，避免链接错误。
  • 线程安全：如果多个线程使用同一个库函数，需要确认该库是线程安全的。
  • 性能与依赖：了解库的依赖关系和潜在性能开销，避免不必要的依赖增加二进制大小。
  • 动态库 vs 静态库：根据应用场景选择使用动态库（减小程序体积）还是静态库（提升独立性）。

Q3. 执行性能如何？

答案：

  • 执行性能依赖于函数调用开销和内存布局的高效性。
  • 使用合适的数据结构和减少不必要的动态分配，可以提高执行效率。
  • 函数调用链过长或频繁的堆分配都会增加执行时间，降低性能。
  • 编译器优化（如内联、常量传播）对执行性能有很大影响。

汇编小结

这里是一些常见的 x86 汇编语法规则，分别解释 AT&T 语法和 Intel 语法的特点，覆盖你提到的所有相关汇编指令。

通用语法概念：

  1. 寄存器：汇编中，CPU 寄存器被直接使用。常见的寄存器有：
    • eax, ebx, ecx, edx：常规数据寄存器。
    • esp：堆栈指针寄存器，用于指向当前栈顶。
    • ebp：基址指针寄存器，常用于函数调用时保存调用栈的基址。
    • eip：指令指针寄存器，指向下一条执行的指令。

AT&T 语法 vs. Intel 语法：

  1. 操作数顺序：

    • AT&T：操作数源, 操作数目标（源在前，目标在后）。
    • Intel：操作数目标, 操作数源（目标在前，源在后）。

  2. 寄存器前缀：

    • AT&T：寄存器前无需前缀，寄存器名是区分大小写的。
    • Intel：寄存器前没有前缀。

  3. 立即数：

    • AT&T：立即数使用 $ 作为前缀，表示常量。
      • 如：$0xc 表示常量 12。
    • Intel：立即数不使用前缀。

  4. 内存地址：

    • AT&T：内存操作数用 () 来表示间接寻址。
      • 如：(%esp) 表示 esp 所指向的地址。
    • Intel：直接写内存地址。
      • 如：[esp] 表示 esp 所指向的地址。

  5. 汇编指令：

    • 指令前缀和后缀：
      • AT&T：指令后可能会带有 b, w, l 后缀，分别表示字节（byte）、字（word）和长字（long word）。
      • Intel：无此后缀，取决于操作数的大小。
    • 示例：
      • AT&T：movl %eax, (%esp) 表示将 eax 中的数据存储到 esp 所指向的内存地址。
      • Intel：mov [esp], eax 同样表示将 eax 中的数据存储到 esp 所指向的内存地址。

常见指令语法

1. mov（数据传输）

  • AT&T：mov source, destination
    • 示例：movl %eax, -0xc(%ebp) 表示将 eax 的内容存到 ebp-0xc 的地址。
  • Intel：mov destination, source
    • 示例：mov [ebp-0xc], eax

2. add（加法运算）

  • AT&T：add source, destination
    • 示例：add $0x10, %esp 表示将 esp 加上 0x10。
  • Intel：add destination, source
    • 示例：add esp, 0x10

3. sub（减法运算）

  • AT&T：sub source, destination
    • 示例：sub $0xc, %esp 表示从 esp 中减去 0xc。
  • Intel：sub destination, source
    • 示例：sub esp, 0xc

4. push（压栈）

  • 将数据压入栈中，栈指针 esp 向下移动。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：push %ebp 表示将 ebp 的内容压入栈中，esp 减少 4 字节（32 位系统）。

5. pop（出栈）

  • 从栈中弹出数据，栈指针 esp 向上移动。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：pop %ebp 表示将栈顶的值弹出，存储到 ebp 中，esp 增加 4 字节（32 位系统）。

6. call（函数调用）

  • 调用一个函数，将当前 eip 寄存器（指向下一条指令的地址）压入栈中，并跳转到指定的函数地址执行。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：call _Z4funci 表示调用名为 _Z4funci 的函数。

7. ret（返回）

  • 从当前函数返回。执行时，恢复先前压入栈中的返回地址到 eip，并跳转到该地址继续执行。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：ret 表示从当前函数返回。

8. lea（加载有效地址）

  • 计算表达式的内存地址，并将该地址加载到寄存器中。
  • AT&T：lea source, destination
    • 示例：lea -0xc(%ebp), %eax 表示将 ebp-0xc 的地址加载到 eax 中。
  • Intel：lea destination, source
    • 示例：lea eax, [ebp-0xc]

9. jmp（跳转）

  • 无条件跳转到指定的地址继续执行。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：jmp _Z4funci 表示跳转到 _Z4funci 函数开始执行。

10. cmp（比较）

  • 比较两个操作数，影响标志寄存器（flags）的状态，通常与条件跳转指令配合使用。
  • AT&T：cmp source, destination
    • 示例：cmp $0, %eax 表示比较 eax 和 0。
  • Intel：cmp destination, source
    • 示例：cmp eax, 0

11. je / jne（条件跳转）

  • 条件跳转，基于前面的 cmp 指令结果：
    • je：等于时跳转（zero flag 置位）。
    • jne：不等于时跳转（zero flag 未置位）。
  • AT&T 和 Intel 都是一样的写法：
    • 示例：je label 或 jne label 表示根据比较结果进行跳转。

==注意：后面的汇编代码采用的都是AT&T语法，最明显的区别即为与之前了解的source,destination的位置都是反过来的，先是source，后是destination==

函数调用

案例

1. 栈帧 (Frame) 的概念

  • 栈帧 (Frame) 是每个函数调用在栈上分配的一段空间，保存函数的参数、局部变量、返回地址等信息。
  • 栈的分配：当一个函数（如func）被调用时，程序会在栈上创建一个新的栈帧。每次调用一个新函数，都会向下扩展栈。
  • 调用关系：main函数调用了func，因此会在栈上为func创建一个新的栈帧。

2. 寄存器 ebp 与 esp 的含义

  1. ebp (Base Pointer)

    • 基址指针寄存器，用于存储当前函数栈帧的基地址（起始地址）。
    • 用途：函数内部的局部变量和参数的地址都是相对于ebp计算的，这样在函数执行过程中可以轻松找到这些变量的位置。
    • 在函数调用时，**原调用者的ebp**也会被保存到当前栈帧，以便函数返回时恢复调用者的状态。

  2. esp (Stack Pointer)

    • 栈顶指针寄存器，指向当前栈的顶端。栈是向下生长的，因此每次压栈时esp会减少，而出栈时esp会增加。
    • 用途：esp用于管理动态内存分配，确保数据按顺序进栈出栈。

3. 栈的结构与增长方向

  • 栈的增长方向：从 高地址向低地址 增长，如图中的红色箭头所示（High 到 Low）。
  • 栈中的每一层栈帧表示一个函数的调用：
    • 上层（高地址）：代表调用者（main）。
    • 下层（低地址）：代表被调用函数（func）。

4. 函数调用时的栈管理过程

  1. main() 调用 func() 的过程：

    • 创建func的栈帧，并将调用者main的ebp存入func的栈帧中。
    • ebp更新为func栈帧的基地址，用于定位func内部的局部变量和参数。
    • esp指向func的栈帧顶端。

  2. 函数返回时：

    • 弹出func的栈帧，恢复main的ebp和esp。
    • 控制权返回到main函数继续执行。

5. 为什么需要使用 ebp 和 esp？

  • 使用**ebp，程序在执行函数内部时可以通过固定的基址指针**访问局部变量和参数。即使esp动态变化，ebp可以保持稳定，方便计算偏移量。
  • 使用**esp**来管理栈顶，确保内存的动态分配和释放。

6. 函数调用的成本 (COST)

  1. 栈帧的创建成本：

    • 每次调用新函数都会在栈上创建一个新的栈帧，并存储调用者的ebp。这个过程会带来一定的开销，尤其是在深度递归时。

  2. 栈帧的销毁成本：

    • 函数返回时，需要弹出栈帧并恢复调用者的寄存器状态，这也会带来开销。

参数传递机制

案例1传参数

  1. Pass by Value (值传递)

解释：

  • 值传递（Pass by Value）：
    • 在调用函数时，传递的是参数的副本。这意味着函数接收到的不是原始变量，而是变量的拷贝。
    • func 函数中的变量 a 和 b 是实参 1 和 2 的副本。对它们的任何修改不会影响原始变量。

调用过程的内存操作：

  1. 函数调用时，将 1 和 2 作为副本传递给 func。

  2. 栈上开辟空间保存 a 和 b 这两个拷贝的值。

  3. 函数体执行：int r = a + b; 计算结果为 3，并返回给 main 函数。

  4. main 函数中的 r 被赋值为 3。

  5. Pass by Reference (引用传递)

解释：

  • 引用传递（Pass by Reference）：
    • 在调用函数时，传递的是变量的引用，即指向原始变量的别名。
    • 在 func 函数中，a 是变量 r 的引用，因此对 a 的任何修改都会直接作用于原变量 r。

调用过程的内存操作：

  1. 调用 func(r) 时，r 的地址会被传递给函数 func。
  2. func 函数中的参数 a 是对 r 的引用，a 和 r 实际上指向同一块内存地址。
  3. 函数体执行：a = 1; 将 r 的值修改为 1。

两种传递方式的区别

特性 Pass by Value Pass by Reference
传递内容 参数的副本 参数的引用（别名）
影响原变量 不会影响 会直接影响原变量
适用场景 数据量小且无需修改的情况 需要修改原始数据的情况
内存消耗 额外分配副本的空间 无需额外空间，使用变量原地址

  • func 函数的栈帧：

4. 栈帧管理与返回地址

  • 为什么使用 ebp：

  • 返回地址：
    call 指令会将返回地址压入栈中。ret 指令会从栈中弹出该地址，并跳回调用函数。
