CPP-2-5

右值

左值：

  • 左值是指具有存储地址的对象，在程序运行时可以通过其地址进行访问。
  • 左值是可以被取地址的对象（& 运算符可以作用于左值）。
  • 左值通常代表在内存中存在的持久性存储（即变量、数组元素、对象等）。
  • 左值可以出现在赋值语句的左侧。

右值：

  • 右值是不能取地址的（& 运算符不能作用于右值）。
  • 右值通常是一些字面量（如 1, 2.5, 'c'）或者是表达式的结果（如 1 + 2，函数返回的临时对象）。
  • 右值不能出现在赋值语句的左侧。

L-value 和 R-value 的关键区别

特性 L-value R-value
存储地址 有存储地址，可以通过指针访问 无存储地址，通常是临时值
可否取地址 可以通过 & 运算符取地址 无法取地址
使用位置 可用于赋值语句左侧 通常是表达式结果或临时对象
生存期 具有较长的生命周期，通常在作用域内有效 生命周期短暂，仅在表达式求值期间有效

const引用可绑右值

  • 非const引用（Non-const references）只能绑定到左值（L-value）。
  • const引用（Const references）既可以绑定到左值，也可以绑定到右值（R-value）。
  • 非const引用无法绑定到右值。
  • 右值引用允许对右值进行修改和操作。
  • 在某些情况下，可以通过右值引用高效地操作临时对象，从而提升性能。

Extern Templates

1. 什么是Extern Templates？

  • 在C++中，模板函数或模板类会在每个使用它们的翻译单元（translation unit）中被实例化。
  • 如果同一个模板在多个翻译单元中被实例化，会导致冗余的代码生成，增加编译时间和最终二进制文件的大小。
  • Extern Templates 是C++11引入的一种机制，允许开发者显式声明某些模板实例化发生在其他翻译单元中，避免重复实例化。

2. 为什么需要Extern Templates？

模板的实例化规则

  • 当模板被使用时（如调用模板函数或实例化模板类对象），编译器会自动生成（实例化）模板代码。
  • 如果一个模板在多个 .cpp 文件中被使用，那么每个文件都会生成该模板的实例代码，这会导致冗余的实例化。

冗余实例化的影响

  1. 编译时间增加： 每个翻译单元都要重新实例化模板。
  2. 可执行文件增大： 每个翻译单元生成的模板实例化代码可能会被重复链接。

解决方法：

  • 使用 Extern Templates，可以显式告诉编译器，某些模板实例化已经在其他翻译单元中完成，不需要在当前翻译单元中再次实例化。

常量表达式

Constant Expressions 概述

内容解析

  1. “Provides more general constant expressions”

    • constexpr 提供了一种更加通用的方式来定义常量表达式，支持在编译期计算复杂的表达式，而不仅仅是简单的数值常量。

  2. “Allows constant expressions involving user-defined types”

    • 在C++11及之后，constexpr支持用户自定义类型的常量表达式（如类、结构体等）。
    • 这使得用户可以编写更加灵活且高效的代码。

  3. “Provides a way to guarantee that an initialization is done at compile time”

    • 使用constexpr关键字可以保证某些初始化在编译时完成，从而提升运行时效率。
    • 如果某个表达式无法在编译时计算，编译器会报错。

运行时效率的提升

  • 所有计算都在编译期完成：
    • 构造点对象。
    • 初始化数组。

  • 运行时无额外开销：
    • 程序运行时直接使用已经计算好的值。

全局总结

1. 什么是constexpr？

  • constexpr是C++11引入的关键字，用于定义可以在编译期计算的常量表达式。
  • 适用于变量、函数、构造函数、类成员等场景。

2. constexpr的作用

  1. 提升性能：
    • 在编译期完成计算，减少运行时开销。
  2. 更灵活的常量表达式：
    • 支持用户自定义类型（如类和结构体）的常量表达式。
  3. 编译期验证：
    • 编译器会验证constexpr表达式是否符合编译期计算的要求。
    • 如果不符合，编译会报错。

3. 使用constexpr的注意事项

  1. 表达式中的所有操作必须是编译期可计算的。
    • 例如，不能使用非constexpr的函数或构造函数。
  2. 使用constexpr函数或构造函数时，输入参数必须是常量。
  3. 在常量表达式中，确保所有变量和函数都支持编译期计算。

4. 实际应用场景

  1. 常量初始化：
    • 定义编译期常量，提升效率。
  2. 模板元编程：
    • 使用constexpr函数简化模板元编程逻辑。
  3. 数组初始化：
    • 使用constexpr数组减少运行时初始化的开销。

3. 捕获方式的作用范围

  1. 按值捕获（[=] 或 [bar]）

    • 捕获的是变量的副本，Lambda 内的修改不会影响原始变量。
    • 适用于只需要读取外部变量，不需要修改的场景。

  2. 按引用捕获（[&] 或 [&, foo]）

    • 捕获的是变量的引用，Lambda 内的修改会反映到原始变量上。
    • 适用于需要修改外部变量的场景。

4. 特殊说明：默认捕获和显式捕获混用

  • 可以混用默认捕获模式和显式捕获：
    • [=, &foo]：按值捕获所有变量，但对 foo 按引用捕获。
    • [&, foo]：按引用捕获所有变量，但对 foo 按值捕获。

5. Lambda 捕获的注意事项

  1. 捕获的变量生命周期

    • 按值捕获的变量是捕获时的副本，与原始变量无关。
    • 按引用捕获的变量要求原始变量在 Lambda 的生命周期内有效。

  2. 可变性（mutable 修饰符）

    • 默认情况下，按值捕获的变量在 Lambda 内是不可修改的。
    • 使用 mutable 关键字可以允许修改按值捕获的变量。

6. 总结

  1. 捕获方式：

    • [ ]：不捕获任何变量。
    • [&]：按引用捕获所有使用的外部变量。
    • [=]：按值捕获所有使用的外部变量。
    • [=, &foo]：按值捕获所有变量，但对 foo 按引用捕获。
    • [bar]：按值捕获指定变量，其余不捕获。

  2. 按值捕获与按引用捕获的选择：

    • 按值捕获： 安全性更高，适合只读操作。
    • 按引用捕获： 更高效，适合需要修改外部变量的场景。

  3. 扩展功能：

    • 使用 mutable 允许按值捕获的变量在 Lambda 内部被修改。
    • C++14 引入的捕获表达式使捕获更加灵活。

1. 什么是委托构造函数？

  • 定义：

  • 特性：
    • C++11 引入。
    • 减少代码重复，方便统一管理初始化逻辑。
    • 提高代码的可维护性。

Uniform统一初始化

1. 什么是 Uniform Initialization？

  • 定义：

    • 统一初始化是C++11引入的一种新语法，用于用大括号 {} 初始化变量或对象。
    • 它提供了一种统一的方式来初始化数组、基本类型、STL容器、自定义类等。

  • 特点：

    • 避免了传统初始化方法的二义性。
    • 支持列表初始化（initializer_list），可以直接对容器或类进行初始化。

2. 对比旧的初始化方式

存在的问题：

  1. 数组初始化是特例：

    • 只有数组可以用 {} 初始化，而其他容器（如std::vector）需要使用成员函数（如push_back）逐个添加元素。

  2. 初始化的二义性：

    • 使用圆括号 () 可能会引发二义性问题：

新风格初始化（New style initialization）

解决的问题：

  1. 统一语法：

    • 无论是数组、STL容器，还是自定义类，都可以使用大括号 {} 进行初始化。

  2. 支持列表初始化：

    • 编译器将 {} 解析为 initializer_list，从而简化了初始化逻辑。

  3. 避免二义性：

    • 使用 {} 初始化时，不会被误解为函数声明。

4. Uniform Initialization 的优点

  1. 语法统一：

    • 使用 {} 统一初始化语法，适用于数组、STL容器、自定义类等。

  2. 列表初始化：

    • 支持通过 initializer_list 直接对对象进行批量初始化，简化代码。

  3. 避免二义性：

    • 使用 {} 初始化不会被误解为函数声明。

  4. 兼容旧代码：

    • 传统的初始化方式仍然有效，新语法是向后兼容的。

5. 注意事项

  1. 优先级：

    • 大括号 {} 初始化的优先级高于其他初始化方式。

    • initializer_list 只能用于构造函数，不能用于成员函数。
    • 其内部数据是只读的，不能修改。

6. 总结

  1. 统一初始化语法：

    • 使用 {} 可以初始化数组、容器、自定义类等，统一了语法规则。

  2. 列表初始化：

    • 编译器将 {} 转换为 initializer_list，提供了简洁的批量初始化方式。

  3. 安全性和可读性：

    • 避免了传统初始化方式的二义性，并禁止了不安全的隐式转换。
